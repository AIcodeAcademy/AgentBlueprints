# 1.1 Assets-Board - Planning - Briefing

**Assets-Board** is a web app for personal use to view financial status at a glance. It focuses on real-time asset valuation with categorization, risk/liquidity flags, and minimalistic data management.

---

## Features

### Must-Have Features (Coding Order):
1. `Add_Asset`  
   - Add assets with: category (e.g., real estate, crypto), quantity, unit value, and auto-calculated "last value" (quantity × unit value).  
2. `Categorize_Assets`  
   - Assign categories with **risk** (e.g., Low/Medium/High) and **liquidity** (e.g., High/Medium/Low) flags.  
3. `View_Asset_Table`  
   - Display assets in a table with columns: Category, Quantity, Unit Value, Last Value, Risk, Liquidity.  
4. `Top_Row_Filters`  
   - Filter assets by category, risk, or liquidity directly from the table header.  

### Nice-to-Have Features:
5. `Export_Import_CSV`  
   - Export/import asset data (e.g., categories, values, flags) in CSV format.  
6. `Summary_Cards`  
   - Display total assets, net worth, and summaries by risk/liquidity.  
7. `Visual_Charts`  
   - Pie charts for asset allocation by category or risk.  

### Won’t-Have Features:
- Transaction history tracking.  
- Collaboration or sharing features.  

---

## Specifications

### User Interface  
1. **`Dashboard_Layout`**  
   - **Table View**: Primary interface with top-row filters for category/risk/liquidity.  
   - **Summary Cards**: Total assets, net worth, and risk/liquidity distribution.  
2. **`Data_Entry_Modal`**  
   - Form to add/edit assets (category dropdown, numeric inputs for quantity/unit value).  
3. **`Minimalist_Design`**  
   - Clean, responsive layout optimized for desktop and mobile.  

### Infrastructure  
4. **`Browser_Local_Storage`**  
   - Persist asset data locally (no backend/server).  
5. **`CSV_Parser`**  
   - Client-side logic for CSV export/import (if implemented).
